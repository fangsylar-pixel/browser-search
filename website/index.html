为什么需要 Browser Search MCP

🔍

真实浏览器搜索

不是调 API 不是爬网页，是启动真正的 Chrome/Edge 去搜索引擎搜结果，不会被封。

🔌

标准 MCP 协议

任何支持 MCP 的客户端都能用 — Ollama、Claude Desktop、Cursor、VS Code。

💰

完全免费

不需要任何 API Key，不依赖第三方搜索服务，自己掌控一切。

⚡

智能缓存

重复搜索自动走缓存，5 分钟 TTL，既快又减少浏览器负担。

🌐

4 大搜索引擎

即装即用

pip install 一条命令装好，自带 MCP 服务器，零配置启动。

工作 流程

LLM / Agent
→
MCP Client
→
browser-search-mcp
→
Chrome/Edge
→
搜索引擎

1. LLM 发出搜索请求 → 2. MCP 协议传输 → 3. 浏览器执行搜索 → 4. 提取结果返回

30 秒 上手

# 1. 安装
pip install browser-search-mcp

# 2. 启动 MCP 服务器
browser-search-mcp

# 3. 在其他工具中配置
# .mcp.json
{
  "mcpServers": {
    "browser-search": {
      "command": "browser-search-mcp"
    }
  }
}

5 个 MCP 工具

工具说明
web_search搜索单个引擎，返回结构化 JSON 结果
web_search_multi同时搜索多个引擎，合并结果
web_search_read_page读取搜索结果页的完整内容
web_search_status检查浏览器、缓存、桥接状态
web_search_discover_browsers扫描本地 CDP 端口发现浏览器
